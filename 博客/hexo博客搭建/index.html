Hexo博客搭建笔记

发表于 2018-06-04 本文字数： 3.4k 标签： 博客 Valine：

hexo是一种静态博客生成工具，可以将markdown文章渲染为html页面，本文主要记录hexo搭建方法以及过程中遇到的问题。

1 依赖软件环境配置

Nodejs：hexo基于nodejs开发并使用npm库分发

Git：一个版本控制系统，hexo主要使用其进行deploy

测试如下↓

2 博客搭建

  • 安装hexo框架：npm install hexo-cli -g

  • 生成博客项目：创建博客文件夹，进入，hexo init初始化目录为hexo项目

  • 安装依赖组件：npm install

这样，一个hexo博客就完整的生成了，使用hexo s运行本地服务器访问http://localhost:4000/查看。

3 博客文件夹

_config.yml：博客配置文件

注意，Hexo有两个config.yml配置文件

  • myblog/_config.yml：常被称为站点配置文件，其中保存着博客站点的相关配置信息，如网站信息、基础插件配置等。
  • myblog/themes/next/_config.yml：常被称为主题配置文件，其中为主题的相关配置，如主题样式、主题插件配置等。

package.json：hexo程序配置信息

scaffolds：模板文件夹，比如其中的post.md是使用hexo new post创建新文章的模板。

source：资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes：主题文件夹，hexo会根据主题生成静态页面。

public：静态文件文件夹，使用hexo g才会生成

4 常用命令

生成、调试、清理、发布

generate · 生成静态文件

当处于hexo项目中时，使用hexo generate或简写命令hexo g生成静态文件，即通过theme将source/_post中的md文件渲染为html放到public文件夹。

常用参数：

  • -d：立即部署
  • -w：修改source文件hexo会显示public的文件变动
  • -f，–force：hexo默认只重新生成变动的文件，使用该参数效果类似hexo clean && hexo generate

server · 启动本地服务器

hexo提供了一个本地服务器以便使用者调试，使用hexo server或简写命令hexo s开启

本地服务器默认开放在4000端口： http://localhost:4000/

clean · 清理缓存

使用hexo clean或简写命令hexo c清除缓存文件 (db.json) 和已生成的静态文件 (public)。

deploy · 网站部署

hexo deploy    # 部署
hexo d    # 简写
hexo d -g    # 部署前先generate

5 切换主题

到主题demo目录 选择一款主题，比如说 Next 主题。

  1. 切换到博客目录，cd myblog
  2. 将主题clone到themes文件夹，git clone https://github.com/iissnan/hexo-theme-next themes/next
  3. 在站点配置文件myblog/_config.yml开启主题：theme: next

使用hexo s运行本地服务器访问http://localhost:4000/查看。

可能会出现注入失败的问题，运行npm install hexo-renderer-swig安装扩展解决。

6 Github Page部署

  1. 在你的github新建一个你github用户名.github.io名称的仓库。

  2. 配置站点_config.yml，比如我的

    # Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
  type: git
  repo: git@github.com:yuandongxu97/yuandongxu97.github.io
  branch: master

  3. hexo d -g部署，这样部署会提示输入github的密码，可以通过配置ssh公钥的方法免密码deploy

  4. 访问 https://yuandongxu97.github.io/

*附录：站点_config.yml配置文件

站点_config.yml配置项详解

网站

参数描述
title网站标题
subtitle网站副标题
description网站描述
keywords网站的关键词。支援多个关键词。
author您的名字
language网站使用的语言。对于简体中文用户来说，使用不同的主题可能需要设置成不同的值，请参考你的主题的文档自行设置，常见的有 zh-Hans和 zh-CN。
timezone网站时区。Hexo 默认使用您电脑的时区。请参考 时区列表 进行设置，如 America/New_York, Japan, 和 UTC 。一般的，对于中国大陆地区可以使用 Asia/Shanghai。

网址

参数描述默认值
url网址, must starts with http:// or https://
root网站根目录
permalink文章的 永久链接 格式:year/:month/:day/:title/
permalink_defaults永久链接中各部分的默认值
pretty_urls改写 permalink 的值来美化 URL
pretty_urls.trailing_index是否在永久链接中保留尾部的 index.html，设置为 false 时去除true
pretty_urls.trailing_html是否在永久链接中保留尾部的 .html, 设置为 false 时去除 (对尾部的 index.html无效)true

博客目录

参数描述默认值
source_dir资源文件夹，这个文件夹用来存放内容。source
public_dir公共文件夹，这个文件夹用于存放生成的站点文件。public
tag_dir标签文件夹tags
archive_dir归档文件夹archives
category_dir分类文件夹categories
code_dirInclude code 文件夹，source_dir 下的子目录downloads/code
i18n_dir国际化（i18n）文件夹:lang
skip_render跳过指定文件的渲染。匹配到的文件将会被不做改动地复制到 public 目录中。您可使用 glob 表达式来匹配路径。

博客文章

参数描述默认值
new_post_name新文章的文件名称:title.md
default_layout预设布局post
auto_spacing在中文和英文之间加入空格false
titlecase把标题转换为 title casefalse
external_link在新标签中打开链接true
external_link.enable在新标签中打开链接true
external_link.field对整个网站（site）生效或仅对文章（post）生效site
external_link.exclude需要排除的域名。主域名和子域名如 www 需分别配置[]
filename_case把文件名称转换为 (1) 小写或 (2) 大写0
render_drafts显示草稿false
post_asset_folder启动 Asset 文件夹false
relative_link把链接改为与根目录的相对位址false
future显示未来的文章true
highlight代码块的设置, see Highlight.js section for usage guide
prismjs代码块的设置, see PrismJS section for usage guide

分类&&标签

参数描述默认值
date_format日期格式YYYY-MM-DD
time_format时间格式HH:mm:ss
updated_option当 Front Matter 中没有指定 updated 时 updated 的取值mtime

分页

参数描述默认值
per_page每页显示的文章量 (0 = 关闭分页功能)10
pagination_dir分页目录page

忽略文件夹

参数描述
includeHexo 默认会忽略隐藏文件和文件夹（包括名称以下划线和 . 开头的文件和文件夹，Hexo 的 _posts 和 _data 等目录除外）。通过设置此字段将使 Hexo 处理他们并将它们复制到 source 目录下。
excludeHexo 会忽略这些文件和目录
ignoreIgnore files/folders
