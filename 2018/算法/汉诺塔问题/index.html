汉诺塔问题

发表于 2018-02-04 本文字数： 1.3k 标签： 算法

11

1. 起源

汉诺塔（又称河内塔）问题是源于印度一个古老传说的益智玩具。大梵天创造世界的时候做了三根金刚石柱子，在一根柱子上从下往上按照大小顺序摞着64片黄金圆盘。大梵天命令婆罗门把圆盘从下面开始按大小顺序重新摆放在另一根柱子上。并且规定，在小圆盘上不能放大圆盘，在三根柱子之间一次只能移动一个圆盘。

2. 数学归纳

有三根相邻的柱子，标号为A,B,C，A柱子上从下到上按金字塔状叠放着n个不同大小的圆盘，要把所有盘子移动到C上，并且每次移动同一根柱子上都不能出现大盘子在小盘子上方，请问至少需要多少次移动，设移动次数为H(n）。

1.1 移动

  • 1个盘子

    盘1，A –> C

  • 2个盘子

    盘1，A –> B

    盘2，A –> C

    盘1，B –> C

  • ···

2.2 归纳

运用归纳法进行归纳，可以将盘子的移动过程归纳为三次移动：

  1. 将n-1个盘子从A借助C移动到B上
  2. 将n号盘移动到C上
  3. 将n-1个盘子从B借助A移动到C上

也就是说总的移动次数H(n)：
$$
H(n) = H(n-1) + 1 + H(n-1)

H(n) = 2^n - 1
$$
由此可见这个问题就是一个递归求解的过程

3. 什么是递归

递归，即自己调用自己，能用递归处理的问题具有两个必要条件

  1. 将较大规模问题能够转化为较小规模的问题。

  2. 存在递归的终止条件。

4. Python实现

逻辑

def move(x, a, b, c):
    if x == 1:
        print("{} --> {}".format(a, c))
    else:
        move(x - 1, a, c, b)
        print("{} --> {}".format(a, c))
        move(x - 1, b, a, c)

move(3, "A", "B" , "C")

# A --> C
# A --> B
# C --> B
# A --> C
# B --> A
# B --> C
# A --> C

完善版本

class TowerOfHanoi:

    def __init__(self):
        self.time = 0

    def move(self, n, a, b):
        # 移动函数
        self.time += 1
        print("第{}次移动：将 {} 号碟从 {} 移动到 {}".format(self.time, n, a, b))
    def hanoi(self, n, a, b, c):
        # 主体逻辑函数
        if n == 1:
            self.move(1, a, c)
        else:
            self.hanoi(n-1, a, c, b)
            self.move(n, a, c)
            self.hanoi(n-1, b, a, c)

t = TowerOfHanoi()
t.hanoi(3, "A", "B", "C")

# 第1次移动：将 1 号碟从 A 移动到 C
# 第2次移动：将 2 号碟从 A 移动到 B
# 第3次移动：将 1 号碟从 C 移动到 B
# 第4次移动：将 3 号碟从 A 移动到 C
# 第5次移动：将 1 号碟从 B 移动到 A
# 第6次移动：将 2 号碟从 B 移动到 C
# 第7次移动：将 1 号碟从 A 移动到 C
