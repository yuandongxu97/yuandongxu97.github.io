《Pro Git》笔记·①Git概述与安装配置

发表于 2020-03-04 本文字数： 1.7k 标签： Git Valine：

git是一种流行的版本控制系统（Version Control System，VCS），

版本控制系统是一种记录一个或若干文件内容变化，以便将来查阅特定版本修订情况的系统。版本控制系统不仅可以应用于软件源代码的文本文件，而且可以对任何类型的文件进行版本控制。用的比较多的如svn，git等。

VCS发展史

  • 本地vcs：简单的本地数据库存储索引
  • 集中式vcs：网络数据库，多方合作化
  • 分布式vcs

Git简史

Git的出现和发展与Linux内核有着密切的关系，Linux 内核开源项目组：

  • 1991-2002年，未使用VCS导致耗费大量时间在提交补丁和保存归档上。
  • 2002-2005年，使用BitKeeper，一个商用的分布式VCS进行代码的管理和维护。
  • 2005年，开发 BitKeeper 的商业公司同 Linux 内核开源社区的合作关系结束,他们收回了 Linux 内核社区免费使用 BitKeeper 的权力。

由此，Linux开源社区开始构想并开发自己的分布式VCS——Git。

Git特点

git使用方法上与CVS、Subversion等非常相似，但实际上git对信息的存储方式和数据认知上与其他VCS有着很大的差异。

特点解释图示
对待数据的方法：记录快照，而非差异基于差异(delta-based) 的版本控制系统(CVS、Subversion、Perforce、Bazaar 等等) 会将它们存储的信息看作是一组基本文件和每个文件随时间逐步累积的差异 。
而Git则是将数据小型文件系统的一系列快照。

Git项目区域划分

  • 工作目录：也就是我们需要进行版本控制的目录。

  • 暂存区：暂存区是一个文件,保存了下次将要提交的文件列表信息,一般在 Git 仓库目录中。 按照 Git 的术语叫做“索引”,不过一般说法还是叫“暂存区”。

  • git目录：工作目录中的.git文件夹，是 Git 用来保存项目的元数据和对象数据库的地方。 这是 Git 中最重要的部分,从其它计算机克隆仓库时,复制的就是这里的数据。

Git工作流程

  1. 在工作区中修改文件。
  2. 将你想要下次提交的更改选择性地暂存,这样只会将更改的部分添加到暂存区。
  3. 提交更新,找到暂存区的文件,将快照永久性存储到 Git 目录。

如果 Git 目录中保存着特定版本的文件,就属于 已提交 状态。 如果文件已修改并放入暂存区,就属于 已暂存 状态。 如果自上次检出后,作了修改但还没有放到暂存区域,就是 已修改 状态。

  • 已修改(modified) ：表示修改了文件,但还没保存到数据库中。

  • 已暂存(staged)：表示对一个已修改文件的当前版本做了标记,使之包含在下次提交的快照中。

  • 已提交(committed)：表示数据已经安全地保存在本地数据库中。

安装方法

  • Windows

    访问 Git Desktop 下载exe安装包进行安装，包含GUI和Bash。

  • Linux

    Debian：sudo dnf install git-all

    Ubuntu：sudo apt install git-all

Git配置

git有三个级别的配置文件，配置项取最高优先级的配置，如 .git/config 的配置变量会覆盖 /etc/gitconfig 中的配置变量。

配置文件控制层次命令行参数优先级
/etc/gitconfig(sudo)、C:\ProgramData\Git\config主机上所有用户及其仓库，系统级--system最低
~/.gitconfig 或 ~/.config/git/config当前用户及其仓库，用户级--global普通
仓库的.git/config当前仓库，仓库级--local最高

查看git的配置及所属配置文件：git config --list --show-origin

设置git身份：

  • 名称：git config --global user.name "John Doe"
  • 邮箱：git config --global user.email johndoe@example.com

设置使用的编辑器，如Vim：git config --global core.editor vim

列出当前使用配置key=value列表：git config --list

获取某项配置：git config user.name

Git帮助

详细手册：git help config

参数手册：git add -h
