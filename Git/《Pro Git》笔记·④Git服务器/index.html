《Pro Git》笔记·④Git服务器

发表于 2020-03-04 本文字数： 975 标签： Git Valine：

虽然远程仓库可以使用如github、gitee等代码托管商，但我们也可以利用网络服务器自己搭建私有仓库。

网络协议

Git 可以使用四种不同的协议来传输资料：本地协议(Local)、HTTP 协议、SSH(Secure Shell)协议、Git协议，以下说明其优缺点及建议用途。

本地协议

基于本地协议的远程版本库为同主机的另一个目录，常见有两种：

  • 每个成员都挂载了一个共享的文件系统，如一个NFS
  • 共用一台电脑（灾难性后果）

使用

git clone /srv/git/project.git
git clone file:///srv/git/project.git
git remote add local_proj /srv/git/project.git

优点是简单，缺点是不够安全

HTTP协议

git的http协议分为两种：

  • 哑 HTTP 协议

    web 服务器仅把裸版本库当作普通文件那样来提供文件服务。优点是设置简单：只需要把一个裸版本库放在 HTTP 根目录,设置一个叫做post-update的hook就可以了

  • 智能 HTTP 协议

    Git 1.6.6引入，让 Git 可以像通过 SSH 那样智能的协商和传输数据，可以使用HTTP 协议的用户名/密码授权（免去ssh设置公钥）

优点是缺点只有一条：在一些服务器上,架设 HTTPS 协议的服务端会比 SSH 协议的棘手一些。除此之外，智能HTTP协议远胜其他协议（但我们最常用的还是ssh）

SSH协议

SSH是架设Git服务器最常用的协议。

git clone ssh://[user@]server/project.git        # URL方法
git clone [user@]server:project.git        # scp方法
# 如果你不指定可选的用户名,那么 Git 会使用当前登录的用的名字。

优势是相对简单，缺点是因为使用ssh所以不支持匿名访问，不利于开源项目。但反着来说，基于ssh更利于公司项目。

Git协议

监听9418port，类似ssh但无需授权，一般个人/小型公司无需使用该协议架设。

服务器搭建

安装git：sudo apt install git-core

创建一个git用户组和用户，用来运行git服务：groupadd git && useradd git -g git

记录ssh登录秘钥到服务器

su git
cd ~
mkdir ~/.ssh && chmod 755 ~/.ssh
