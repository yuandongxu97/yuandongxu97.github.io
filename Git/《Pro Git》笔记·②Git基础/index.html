《Pro Git》笔记·②Git基础

发表于 2020-03-04 本文字数： 3.4k 标签： Git Valine：

本章主要学习Git的基础知识，包括但不限于：

  • 配置并初始化一个仓库(repository)
  • 开始或停止跟踪(track)文件、暂存(stage)或提交(commit)更改
  • 如何配置 Git 来忽略指定的文件和文件模式
  • 如何迅速而简单地撤销错误操作
  • 如何浏览你的项目的历史版本以及不同提交(commits)之间的差异
  • 如何向你的远程仓库推送(push)以及如何从你的远程仓库拉取(pull)文件。
  • Git command alias

获取Git仓库

git init创建本地仓库

git init：将所处目录转化为git仓库。

git init gittest：在当前目录创建一个gittest文件夹并将其转化为一个git仓库。

git clone克隆远程仓库到本地

git clone <url> [floder_name]命令会将远程仓库克隆到本地。

其会在当前目录会生成一个新目录，如果clone时不指定floder_name的话会默认使用项目名称，并在这个目录下初始化一个 .git 文件夹, 从远程仓库拉取下所有数据放入 .git 文件夹,然后从中读取最新版本的文件的拷贝放到文件夹中。

git 支持多种协议的URL：

  • https://
  • git协议，git://
  • SSH 传输协议，如 user@server:path/to/repo.git

追踪与状态查看

仓库文件状态

git的作用是版本控制即跟踪文件变化，因此一个git仓库中的文件会有两种状态：

  • 已跟踪（tracked）：被版本控制的文件，未修改、已修改、已暂存、已提交这些被git知道状态的文件都属于这个状态，如初次克隆仓库时仓库中的文件都属于tracked未修改状态。
  • 未跟踪（untracked）：工作目录中除已跟踪文件外的其它所有文件都属于未跟踪文件，不存在于git的任何快照历史中。

git status查看文件状态

git status是git中常用的命令，作用为列出当前工作目录中文件的状态，包括：

  • 未忽略文件中的已跟踪和未跟踪
  • 已跟踪文件的状态

git status得到的信息较为繁琐，使用 git status -s 命令或 git status --short 命令,你将得到一种格式更为紧凑的输出。命令输出中有两栏,左栏指明了暂存区的状态,右栏指明了工作区的状态

  • 新添加的未跟踪文件， ?? 标记
  • 新添加到暂存区中的文件，A 标记
  • 修改过的文件， M 标记。
  • 添加到暂存区后又被修改的文件，AM。

git add 追踪文件、暂存修改

  • 对于未跟踪的文件：开始跟踪文件，并将文件快照保存至暂存区
  • 对于已跟踪的文件：将文件快照保存至暂存区

可以使用git add <filename1> <filename2> ···指定文件名称的使用，或者使用git add .添加所有

git commit提交到版本库

git commit：将暂存区提交到版本库

git commit -m "备注信息"：将暂存区提交到版本库并添加提交备注

git commit -a：暂存所有已跟踪文件并提交

.gitignore配置忽略文件

如果仓库中有一直不需要跟踪的文件，我们可以通过配置.gitignore文件告知仓库不需要跟踪这些文件，如使用git add .也会忽略。

  • 仓库根目录的.gitignore：递归地应用到整个仓库中。
  • 子文件夹中的.gitignore：只作用于它所在的目录中。

配置规范：

规范配置实例说明
以#开头的注释行会被忽略# this is a comment
使用glob（shell正则）递归匹配*.[oa]会忽略任何目录下的.a和.o文件
以/结尾标识目录build/忽略任何目录下的build文件夹
绝对路径模式防止递归/test会忽略仓库根的test，但不会忽略其他子文件夹中的test
使用!排除例外!lib.a前面忽略了任何.a文件，但这项配置从中排除了所有的lib.a
目录递归doc/**/*.pdf忽略所有的doc/ 目录及其所有子目录下的 .pdf 文件

git diff查看差异

文件与暂存区差异，即未暂存的差异：git diff

暂存区与最后一次提交的差异，即未提交差异：git diff --staged

移除文件

删除已追踪、未暂存的本地文件：git rm <file>

删除已追踪、已修改/已暂存的本地文件和暂存库文件：git rm -rf <file>

移除已跟踪、但不删除本地文件：git rm --cached

移动文件

$ git mv README.md README
# 相当于
$ mv README.md README
$ git rm README.md
$ git add README

commit历史

git log会显示仓库中每次commit的节点信息

使用-p或--patch会以补丁形式显示每次提交的差异

使用-number限定深度

--stat 简略的总结

会显示在每次commit的下面列出所有被修改过的文件、有多少文件被修改了，以及被修改过的文件的哪些行被移除或是添加了

--pretty=输出格式控制

  • git log --pretty=oneline

    每个commit信息放在一行

  • git log --pretty=format:"%h %s" --graph

    指定格式

操作撤销

补提交

git commit --amend

某次commit后发现漏掉了文件没有添加或者备注信息错误就可以使用该命令进行补充提交操作，会将当前暂存区提交到上个commit head上，如果暂存区为空，那么就是只修改备注信息。

或者说是：取消上一次commit并重新commit暂存区

取消stage

git restore --staged <file>s

如我们有a、b两个文件需要分开commit，但是错误的使用git add *将他们都添加到了暂存区，如何取消一个暂存呢？

撤销工作区修改

git restore <file>s

拉取最近的commit到stage

拉取最近一次提交到版本库的文件到暂存区，不影响工作区 ：git reset HEAD -- <file>

拉取stage到工作区

把暂存区文件拉到工作区：git checkout -- <file>

远程仓库使用

对于git来说远程仓库指非当前目录的git仓库，而非特指互联网上的仓库。

修复ssh访问github brokenpipe，添加到信任主机列表：ssh-keyscan -t rsa github.com >> ~/.ssh/known_hosts

假设我们在github有一个仓库：git@github.com:yuandongxu97/mycode.git

两种关联远程仓库到本地的方法

  • clone远程仓库到本地，这样这个仓库就直接remote指向远程仓库

  • 本地init一个空仓库，remote add指向一个远程仓库，然后fetch拉取仓库内容

remote 本地与远程关联

显示关联的远程仓库简称：git remote

显示关联的远程仓库简称与URL：git remote -v

显示仓库的详细信息：git remote show <alias>

关联远程到本地：git remote add <alias> <URL>

alias重命名：git remote rename <old> <new>

移除远程关联：git remote rm <alias>

push 推送本地到远程

将分支推送到alias的branch：git push [alias] [branch]

fetch 下载差异数据

拉取alias仓库branch分支到本地：git fetch <alias> <branch>s

merge 分支合并

将指定分支合并到当前分支：git merge <alias>/<branch>

pull 拉取合并

从远程拉取数据并尝试合并到当前所在的分支：git pull alias/branch branch

Git Command alias

比如使用git unstage作为git reset HEAD --别名

git config --global alias.unstage 'reset HEAD --'

git unstage fileA
# 等于
git reset HEAD -- fileA

又比如：查看最后一次提交，git config --global alias.last 'log -1 HEAD'
