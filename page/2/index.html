Linux基于zplug快速部署zshell终端环境

zsh的全拼为z shell，是一种类似Bash shell的命令解释器，和bash相比，zsh可以通过配置插件大大提高效率。

zplug和我们一开始接触zsh使用的oh-my-zsh都属于zsh的插件管理器，两者有各自的优缺点：

  • oh-my-zsh：上手容易，即开即用，但插件管理上更倾向于手动管理。
  • zplug：自动化的插件管理，用户只需要关注zshrc配置文件即可，可实现通过一份配置文件多终端快速部署相同环境。

如何基于Linux自搭建代理服务器

发表于 2020-05-04 本文字数： 2.7k 标签： Linux Valine：

爬虫因为网站限制的原因经常用到ip代理，解决方案无非几种：

  • 网络上寻找一些免费代理，优点：免费不限量；缺点：可用性较低，验证费时间费资源。一些有免费代理的网站，西刺代理，站大爷，89免费代理等等，网上可以搜出一大堆。
  • 购买代理ip，和找免费的差不多，一般有免费代理ip的网站基本都有收费的api。优点：可用性较高，较稳定。缺点：要钱。严格说，这个也不算缺点，要钱很多人也能接受，带来的方便还是很大程度的。
  • 自己搭建ip代理，自己租服务器自己搭。优点：稳定，可控。缺点：麻烦，成本高。

其实代理自己整一台服务器搭一下就可以，就是其中曲折有点多···，搭建代理也有很多种协议，工具，都实验过后选择使用tinyproxy，简单方便。

Python标准库·日志logging

发表于 2020-04-05 本文字数： 9.2k 标签： Python标准库 Valine：

日志是用来记录程序在运行过程中发生的状况，在程序开发过程中添加日志模块能够帮助我们了解程序运行过程中发生了哪些事件，相比于使用print输出信息，使用logging功能更多、信息更为规范。

标准日志级别根据事件的轻重分为：debug（调试）、info（工作）、warning（异常）、error（功能错误）、critical（程序错误）

logging使用

因为logging默认的日志级别为warning，所以仅输出大于等于warning的日志（debug、info）

import logging

logging.debug('This is debug message')
logging.info('This is info message')
logging.warning('This is warning message')
logging.error('This is error message')
logging.critical('This is critical message')

# WARNING:root:This is warning message
# ERROR:root:This is error message
# CRITICAL:root:This is critical message

AJAX，Asynchronous JavaScript and XML，一种使用异步的 JavaScript 和 XML创建动态网页的技术，通过在后台与服务器进行少量数据交换，使网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新。

ajax请求在前端中有两种表现形式，一种是在后台偷偷获取数据，另一种是在右侧进度条滑动到底部时进行数据的加载。

Ajax并不是一种新技术，而是使用了一些原有的internet标准组合而成，所以和平台/浏览器内核无关

  • 前端显示、用户交互：javascript、DOM
  • 异步的与服务器交换数据：XMLHttpRequest 对象
  • 给数据定义样式：css
  • 数据格式：xml、json等
