Deb系包管理工具的使用

发表于 2020-04-05 本文字数： 1.3k 标签： Linux Valine：

Linux软件管理系统的诞生之路：

  • 最初，Linux想安装软件只有编译.tar.gz的打包文件
  • 如何管理主机上已安装的软件包，于是本地包管理工具dpkg诞生了，继而诞生redhat的rpm。
  • 新的需求，GNU/Linux需要一个快速、实用、高效的方法来安装软件包，当软件包更新时，这个工具应该能自动管理关联文件和维护已有配置文件。Debain通过开发dpkg的前端工具apt，一个基于客户机/服务器的软件包管理工具解决了这个问题。

apt作为一个软件包，由 apt-get、apt-cache、apt-cdrom等命令组成，通过 /etc/apt/sources.list配置软件源地址，更改软件信息（增删改）需要root权限。

虽然apt是dpkg的前端，但apt也是aptitude等管理程序的后端实现。

Docker容器是一种虚拟化技术，基于google的golang语言开发，在github进行开源，底层原理使用了Linux容器的LXC技术，并对其进行了进一步的封装。

LXC技术为将Linux进程沙盒化，使得进程之间相互隔离，并且能够控制各进程的资源分配。

与传统的虚拟化技术如VMware虚拟机等相比：

方面docker容器虚拟机
虚拟化层面模拟物理机，硬件层面基于操作系统层，软件层面
隔离级别操作系统进程
部署与启动秒级分钟级
磁盘使用基于同一个OS内核，200M以内一个虚拟机相当于一个完整的物理机，GB级
单机实例数量数千几十
宿主机系统资源/个0~0.050.1~?

应用上：

  • 虚拟机更擅长于彻底隔离整个运行环境，如云服务提供商通常采用虚拟机技术隔离不同的用户。

  • Docker通常用于隔离不同的应用，例如前端，后端以及数据库。

Docker网络配置笔记

发表于 2020-08-11 本文字数： 3.9k 标签： Docker Valine：

一、前言

        Docker使用Linux桥接在宿主机虚拟一个Docker容器网桥(docker0)，启动一个容器时会根据Docker0的网段分配给容器一个内部IP地址，称为Container-IP，同时Docker网桥是每个容器的getway。因为在同一宿主机内的容器都接入同一个网桥，这样容器之间就能够通过容器的Container-IP直接通信。

        Docker网桥是宿主机虚拟出来的，并不是真实存在的网络设备，外部网络是无法寻址到的，这也意味着外部网络无法通过直接Container-IP访问到容器。如果容器希望外部访问能够访问到，可以通过映射容器端口到宿主主机（端口映射），即docker run创建容器时候通过 -p 或 -P 参数来启用，访问容器的时候就通过[宿主机IP]:[容器端口]访问容器。

        网桥模式仅为docker的一种网络模式，本文主要记录Docker提供的四种网络模式原理与使用。

Python标准库·异步并发asyncio

发表于 2020-07-06 本文字数： 2.7k 标签： Python标准库 Valine：

我们日常使用的Python版本为CPython，而CPython有一个GIL(全局解释器锁)的设计。

GIL类似操作系统的Mutex（互斥锁），功能为保证在任一时刻只能有一个解释器线程在运行，因此Python的多线程是一个运行、其他挂起的伪并发。

使用协程的优点：

  • 协程极高的执行效率。因为子程序切换不是线程切换且由程序自身控制，因此，没有线程切换的开销，多线程任务的线程数量越多，协程的性能优势就越明显。
  • 不需要多线程的锁机制，因为只有一个线程，也不存在同时写变量冲突，在协程中控制共享资源不加锁，只需要判断状态就好了，所以执行效率比多线程高很多。
  • 因为协程是一个线程执行，那怎么利用多核CPU呢？最简单的方法是多进程+协程，既充分利用多核，又充分发挥协程的高效率，可获得极高的性能。

asyncio是Python中的一个基于协程实现异步高并发的库，3.4版本开始引入到标准库，3.5添加了 async/await 语法进行协程的实现，其被用作多个提供高性能 Python 异步框架的基础，包括网络和网站服务，数据库连接库，分布式任务队列等等，它往往是构建 IO 密集型和高层级 结构化 网络代码的最佳选择。

Python命名空间及作用域

发表于 2020-06-04 本文字数： 2k 标签： Python Valine：

命名空间（namespace）- 名称存储

官方文档说明：

A namespace is a mapping from names to objects.Most namespaces are currently implemented as Python dictionaries。

命名空间是从名称到对象的映射，大部分的命名空间都是通过 Python 字典来实现的。

命名空间其实就是Python对象名称的保存位置，各个命名空间是独立的，因此两个相互独立的函数中可以各有一个a变量，不同的命名空间中的对象名称是可以重名而没有任何影响。

作用域（scope）- 名称查找

官方文档说明：

A scope is a textual region of a Python program where a namespace is directly accessible. “Directly accessible” here means that an unqualified reference to a name attempts to find the name in the namespace.

作用域即代码可以直接访问命名空间的区域

python中对象不是哪个位置都可以访问的，访问权限决定于这个对象是在哪里创建的，假设你在X处可以访问到A命名空间的变量，那么X就是A的作用域之一，直接访问一个变量，会从内到外依次访问所有的作用域直到找到，否则会报未定义的错误。
