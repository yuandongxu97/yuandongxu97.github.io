Deb系包管理工具的使用

发表于 2020-04-05 本文字数： 1.3k 标签： Linux Valine：

Linux软件管理系统的诞生之路：

  • 最初，Linux想安装软件只有编译.tar.gz的打包文件
  • 如何管理主机上已安装的软件包，于是本地包管理工具dpkg诞生了，继而诞生redhat的rpm。
  • 新的需求，GNU/Linux需要一个快速、实用、高效的方法来安装软件包，当软件包更新时，这个工具应该能自动管理关联文件和维护已有配置文件。Debain通过开发dpkg的前端工具apt，一个基于客户机/服务器的软件包管理工具解决了这个问题。

apt作为一个软件包，由 apt-get、apt-cache、apt-cdrom等命令组成，通过 /etc/apt/sources.list配置软件源地址，更改软件信息（增删改）需要root权限。

虽然apt是dpkg的前端，但apt也是aptitude等管理程序的后端实现。

apt常用命令

  • 更新sources.list源数据：apt update
  • 更新所有软件包：apt upgrade（-u：显示待更新列表）
  • 搜索软件包：apt search packagename
  • 显示软件包的详细信息：apt show packagename
  • 安装软件包：apt install packagename
  • 卸载已安装的软件包（保留配置文档）：apt remove packagename
  • 卸载已安装的软件包（删除配置文档）：apt remove --purge packagename
  • 删除软件包及其依赖：apt autoremove packagename
  • 清除已卸载软件的备份：apt autoclean

apt其他命令

  • 显示依赖组成：apt-cache depends packagename
  • 下载源码：apt source packagename
  • 修复dpkg安装本地包时缺少的依赖（不指定包名称!）：apt install -f

dpkg常用命令

  • 安装本地包：dpkg -i path/package.deb

  • 批量安装：dpkg -R /usr/local/src

  • 已安装包列表：dpkg -l

  • 删除软件包（保留其配置信息）：dpkg -r

  • 删除一个包（包括配置信息）：dpkg -P

  • 版本差异：dpkg –compare-versions ver1 op ver20

  • 包安装后目录信息：dpkg -L

关闭upgrade命令中的内核更新

升级内核有好处也有坏处，好处略，坏处会造成依赖特定内核的软件发生error。。。，以下为ubuntu apt upgrade忽略内核更新的方法。

正常apt安装的软件信息为

设置为hold状态的软件即不会被更新

设置hold标志方法

uname -r # 查询内核版本
# 5.4.0-26-generic
dpkg --get-selections | grep linux-image    # 查询本地内核包有哪些
# linux-image-5.4.0-26-generic                       install
# linux-image-5.4.0-51-generic                       install
echo "linux-image-5.4.0-26-generic hold" | dpkg --set-selections    # 指定hold标志

apt remove卸载不需要的内核，如果需要安装最新内核，使用apt search + apt install 即可。
