Linux权限提升命令sudo的使用与配置方法

发表于 2019-03-04 本文字数： 1k 标签： Linux Valine：

Linux是基于多用户多任务处理的操作系统，通常只用root用户完全的控制权限，其他一些非root用户只有特定的权限，如果普通用户需要执行超出自身权限的命令，则需要进行权限提升。

sudo就是解决这个需求的命令，root在进行配置后允许特定的用户在不知道root密码的情况下，通过将命令前添加sudo前缀即可执行root权限操作。比如最常见的用法，以普通用户的身份安装vim软件：sudo apt install vim

sudo命令的执行过程

  1. 当用户执行sudo时，系统会主动寻找/etc/sudoers文件，判断该用户是否有执行sudo的权限
  2. 确认用户具有可执行sudo的权限后，让用户输入用户自己的密码确认
  3. 若密码输入成功，则开始执行sudo后续的命令

注意：这个密码会保持15分钟的有效期，有效期内重复使用sudo不会重复要求输入密码，新的终端/超过五分钟都会要求重新输入密码

配置文件详解

sudo默认的配置文件为/etc/sudoers，以我的ubuntu为例，有以下这三条默认配置项

sudoers配置语法如下，中括号为可选项，省略即默认：

授权用户/组 主机=[(切换到哪些用户或组)] [是否需要输入密码验证] 命令1,命令2,...

  1. 授权用户/组：那个组或者那个用户的配置，上面的root用户、sudo和admin组。
  2. 主机：允许使用sudo的主机，ALL表示所有。
  3. (切换到哪些用户或组)：默认为(root:root)表示允许提权到root，(ALL:ALL)表示允许提权到任意用户（如其他用户）。
  4. 是否需要输入密码验证：默认需要密码验证，可通过配置 NOPASSWD:免密码验证。
  5. 命令序列：即允许使用的命令列表，ALL表示允许所有命令，使用*通配、!反选。

注意命令序列中的命令需要写成绝对路径的形式以保证安全！

赋予用户sudo操作权限

通过useradd添加的用户并不具备sudo权限，方法一是将用户添加到sudoers存在的组

usermod -a -G sudo yuan    # 将yuan添加到sudo组

或者直接配置sudoers添加用户的配置即可

yuan    ALL=(ALL:ALL) ALL

sudo免密码

比如我要将yuan用户的sudo执行配置免密码。

  1. 使用groups查看用户所处的组列表，发现其中有sudo组

  2. 编辑sudoers配置文件，追加一行yuan的配置，并修改sudo组的配置防止组配置覆盖用户配置

其他配置

  • 输入密码有反馈：配置文件添加一行 Defaults pwfeedback
  • 修改密码有效期时长：配置文件添加一行 Defaults timestamp_timeout=60，单位min

查询用户的sudo权限：sudo -l
