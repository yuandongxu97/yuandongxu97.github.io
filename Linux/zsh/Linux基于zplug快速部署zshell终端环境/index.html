Linux基于zplug快速部署zshell终端环境

发表于 2020-05-06 本文字数： 2.1k 标签： Linux Valine：

zsh的全拼为z shell，是一种类似Bash shell的命令解释器，和bash相比，zsh可以通过配置插件大大提高效率。

zplug和我们一开始接触zsh使用的oh-my-zsh都属于zsh的插件管理器，两者有各自的优缺点：

  • oh-my-zsh：上手容易，即开即用，但插件管理上更倾向于手动管理。
  • zplug：自动化的插件管理，用户只需要关注zshrc配置文件即可，可实现通过一份配置文件多终端快速部署相同环境。

1. 第一次配置

以ubuntu为例，Zplug Github项目地址

环境配置

  1. 安装zsh：sudo apt install zsh
  2. 使用chsh -s /bin/zsh切换用户默认shell为zsh，可以使用cat /etc/shells查看系统中所有已安装的shell，配置成功后重新打开terminal即可切换至zshell。
  3. 将zplug安装到用户根目录下：git clone https://github.com/zplug/zplug ~/.zplug

配置文件

前置配置完成后，我们已经安装好了zsh和zplug，且有了一份zshrc配置文件，每次打开zsh terminal都会重载zshrc这个配置文件。

以下是我的zshrc中关于zplug的配置：

# Zplug
if [[ -f ~/.zplug/init.zsh ]] {
    source ~/.zplug/init.zsh

    # plugin list
    zplug "agnoster/agnoster-zsh-theme", as:theme, depth:1      # 主题
    zplug 'zplug/zplug', hook-build:'zplug --self-manage'
    zplug "zsh-users/zsh-completions"
    zplug "zsh-users/zsh-autosuggestions"
    zplug "zsh-users/zsh-history-substring-search"
    zplug "zsh-users/zsh-syntax-highlighting", defer:2
    zplug "plugins/sudo", from:oh-my-zsh
    zplug "plugins/git", from:oh-my-zsh
    zplug "zpm-zsh/ls"
    zplug "plugins/docker", from:oh-my-zsh
    zplug "plugins/composer", from:oh-my-zsh
    zplug "plugins/extract", from:oh-my-zsh

    # check uninstall
    if ! zplug check --verbose; then
        printf "Install? [y/N]: "
        if read -q; then
            echo; zplug install
        fi
    fi

    # Then, source plugins and add commands to $PATH
    zplug load
}

关于这个shell函数详细解释：

  • if [[ -f ~/.zplug/init.zsh ]]：检测zplug是否安装，是则运行{}代码块

  • source ~/.zplug/init.zsh：启动zplug

  • # plugin list：我们需要使用的插件列表，写法参考官方说明：

  • # check uninstall：使用zplug的zplug check --verbose检测是否存在未安装的插件，并根据用户输入决定是否安装未安装插件。

  • zplug load：重载zplug使插件生效

配置zshrc完成后，使用source ~/.zshrc重载zsh即可使这些配置手动生效。

2. 其他终端部署

  1. 将已配置好的zshrc复制一份到用户目录根
  2. sudo apt install zsh && chsh -s /bin/zsh
  3. git clone https://github.com/zplug/zplug ~/.zplug
  4. source ~/.zshrc，然后输入y回车，完成部署

3. 插件相关

插件的安装位置

安装位置为ZPLUG_HOME/repos下以安装源分类，如agnoster主题

zsh-autosuggestions的快捷键冲突

我是将接受建议的按键绑定为,键，zshrc添加一行：bindkey ',' autosuggest-accept%

4. 安装中易出现的问题

agnoster图形字体问题

安装使用powerline字体即可，或者使用consolas也可以使大部分正常显示。

perl: warning: Falling back to a fallback locale (“en_US.UTF-8”)

语言包没装全，运行sudo apt install locales-all安装即可解决。
