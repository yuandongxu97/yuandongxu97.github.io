Django(一)·环境搭建与项目初始化

发表于 2020-05-05 本文字数： 2.8k 标签： Django

Django是一个用Python编写的Web框架。

Web框架是一种软件，基于web框架可以开发动态网站，各种应用程序以及服务。它提供了一系列工具和功能，可以解决许多与Web开发相关的常见问题，比如：安全功能，数据库访问，会话，模板处理，URL路由，国际化，本地化，等等。

Django提供了各种各样的开箱即用的功能，比如：

  • 用于开发和测试的独立Web服务器

  • 缓存

  • 中间件系统

  • ORM

  • 模板引擎

  • 表单处理

  • 基于Python单元测试的工具接口。

Django还自带内部电池，提供内置应用程序，比如一个认证系统，一个可用于 CRUD(增删改查) 操作并且自动生成页面的后台管理界面，生成订阅文档（RSS/Atom）,站点地图等。甚至在django中内建了一个地理信息系统（GIS）框架。

1. Django环境搭建

搭建一个完整的django环境流程包括：

  • 安装python
  • 创建虚拟环境venv
  • 安装django框架

虽然使用虚拟环境不是强制性的，但使用虚拟环境可以有效的避免一些问题。当使用 Django 开发一个网站或者一个Web项目的时候，不得不安装外部库以支持开发是非常常见的事情。使用虚拟环境，你开发的每个项目都会有其独立的环境。这样的话，包之间的依赖关系不会发生冲突。同时也使得你能在不同Django版本上运行的本地机器的项目。

1.1 安装Python

安装Python非常的简单，到 Python ftp分发 下载对应平台的版本，进行安装后将安装位置的bin目录添加到环境变量即可！

1.2 创建虚拟环境

python3.3以下版本可以安装 vitualenv包进行虚拟环境的创建

pip install virtualenv
mkdir django_test
cd django_test
virtualenv django_venv

Python3.3以上版本可以直接使用内置包 venv进行虚拟环境的创建。

mkdir django_test
cd django_test
python -m venv django_venv

如果需要指定Python版本

virtualenv -p /usr/bin/python2.7 my_project_env

激活/关闭环境

source my_project_env/bin/activate # linux/mac
./my_project_env/bin/activate    # windows
deactivate    # close

1.3 安装django

2. 项目初始化

2.1 项目创建

使用django自带的命令行工具创建一个新项目，命令行工具django-admin会在安装Django的时候一起自动安装好。

django-admin startproject myproject

创建完成后的项目文件夹结构

django_test/                <-- 项目容器文件夹
 |-- myproject/             <-- Django项目文件夹
 |    |-- myproject/
 |    |    |-- __init__.py
 |    |    |-- settings.py
 |    |    |-- urls.py
 |    |    |-- wsgi.py
 |    +-- manage.py
 +-- django_venv/           <-- 虚拟环境

我们最初的项目结构由五个文件组成：

  • manage.py：使用django-admin命令行工具的快捷方式。它用于运行与我们项目相关的管理命令。我们将使用它来运行开发服务器，运行测试，创建迁移等等。

  • __init.py：将文件夹声明为一个Python包

  • settings.py：这个文件包含了所有的项目配置。

  • urls.py：负责映射我们项目中的路由和路径。

  • wsgi.py：用于部署的简单网关接口。

2.2 服务器测试

django还自带了一个简单的网络服务器。在开发过程中非常方便，所以我们无需安装任何其他软件即可在本地运行项目。我们可以通过执行命令来测试一下它：

python manage.py runserver

浏览器访问：**http://127.0.0.1:8000**，即可看到如下页面，使用 ctrl+C停止服务器。

2.3 Django应用

django项目代码组织主要遵循两个原则：

  • django中某件事应该由应用程序也就是app进行完成，app通常由一组models(数据库表)**，views(视图)，templates(模板)，tests(测试)** 组成。
  • 项目是配置和应用程序的集合，一个Django项目可以由多个应用程序或一个应用程序组成。
2.3.1 创建app

在项目中创建一个新的app

python .\manage.py startapp boards
# 或者
django-admin startapp boards

新的项目结构

myproject/
 |-- myproject/
 |    |-- boards/                <-- 我们新的Django应用（app）!
 |    |    |-- migrations/
 |    |    |    +-- __init__.py
 |    |    |-- __init__.py
 |    |    |-- admin.py
 |    |    |-- apps.py
 |    |    |-- models.py
 |    |    |-- tests.py
 |    |    +-- views.py
 |    |-- myproject/
 |    |    |-- __init__.py
 |    |    |-- settings.py
 |    |    |-- urls.py
 |    |    |-- wsgi.py
 |    +-- manage.py
 +-- venv/

在app中每个文件的作用为：

  • migrations/**：在这个文件夹里，Django会存储一些文件以跟踪你在models.py文件中创建的变更，用来保持数据库和models.py**的同步。
  • admin.py：这个文件为一个django内置的应用程序Django Admin的配置文件。
  • apps.py：这是应用程序本身的配置文件。
  • models.py：这里是我们定义Web应用程序数据实例的地方。models会由Django自动转换为数据库表。
  • tests.py：这个文件用来写当前应用程序的单元测试。
  • views.py：这是我们处理Web应用程序请求(request)/响应(resopnse)周期的文件。
2.3.2 启用app

当我们完成了某个app的设计后，需要在django项目中启用该app。

打开settings.py找到INSTALLED_APPS变量，这个变量控制django注册的app信息，Django项目默认已经安装了6个内置应用程序。它们提供大多数Web应用程序所需的常用功能，如身份验证，会话，静态文件管理（图像，JavaScript，CSS等）等。

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    # user app
    'boards',
]
