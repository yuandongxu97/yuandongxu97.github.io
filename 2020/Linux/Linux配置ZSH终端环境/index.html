Linux配置ZSH终端环境

发表于 2020-01-04 本文字数： 1.6k 标签： Linux

ZSH是mac的默认终端，相当于Linux发行版的bash，本文主要记录了Linux中zsh+onmyzsh的配置方法及优化过程，zsh优点：

1.完全兼容bash,之前bash下的使用习惯,shell脚本都可以完全兼容。

2.更强大的tab补全,当你切换目录敲两下tab,他可以列出当前目录下面的所有目录,…

3.更智能的切换目录,比如你要进入一个很深的目录，zsh会帮助你完成路径的补全

4.命令选项补齐,比如输入yum,然后按tab,即可显示出yum都有哪些命令选项。

5.命令参数补齐,比如要kill一个进程,直接输入kill 进程名,会自动显示出进程的pid

1. ZSH的安装

1.1 安装zsh

如果你用 Redhat Linux，执行：sudo yum install zsh
如果你用 Ubuntu Linux，执行：sudo apt-get install zsh

查看系统中的shell版本

cat /etc/shells

切换系统默认shell为zsh

chsh -s /bin/zsh

1.2 安装 ohmyzsh

ohmyzsh的GitHub项目地址

自动安装：

wget https://github.com/robbyrussell/oh-my-zsh/raw/master/tools/install.sh -O - | sh

手动安装：

git clone git://github.com/robbyrussell/oh-my-zsh.git ~/.oh-my-zsh 
cp ~/.oh-my-zsh/templates/zshrc.zsh-template ~/.zshrc

或者前两种方式都不合适，可以尝试此种：

1. 到项目仓库下载zip包
2. 安装unzip
3. unzip master.zip
4. cd cd ./ohmyzsh-master/tools/ && sh install.sh

都不复杂，安装完成之后退出当前会话重新打开一个终端窗口，你就可以见到彩色的提示了

2. ZSH的优化

zsh 的配置在~/.zshrc文件，用 vim打开.zshrc进行配置，配置项如

查看zsh的主题：echo $ZSH_THEME

ZSH_THEME="agnoster"        # 主题

2.1 zsh隐藏终端用户名和计算机名

由于oh my zsh会经常更新，为了防止ohmyzsh更新个人配置被覆盖，将本地的主题文件复制出一份

cp ~/.oh-my-zsh/themes/agnoster.zsh-theme ~/.oh-my-zsh/themes/agnoster-my.zsh-theme

修改主题文件

vim ~/.oh-my-zsh/themes/agnoster-my.zsh-theme

注释末尾配置

# Main prompt
build_prompt() {
  RETVAL=$?
  prompt_status
  prompt_virtualenv
 # prompt_context
  prompt_dir
  prompt_git
  prompt_hg
  prompt_end
}

修改~/.zshrc

ZSH_THEME="agnoster-my"

立即生效环境变量

source ~/.zshrc

3. 插件

插件放置文件夹：${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins

插件启用：~/.zshrc的

# plugins
plugins=(
  zsh-history-substring-search
  sudo
  git
)

sudo

自带插件。按两下 ESC 键，自动为命令前加上 sudo。

zsh-autosuggestions

这个插件会根据历史输入指令的记录即时的提示

git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

zsh-syntax-highlighting

这是一个命令高亮插件，输入为绿色时表示正确的命令，路径带有下划线时表示可用路径

git clone https://github.com/zsh-users/zsh-syntax-highlighting.git \
${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

git-open

在git目录打开相应的github页面

git clone https://github.com/paulirish/git-open.git $ZSH_CUSTOM/plugins/git-open

thefuck

输入 fuck 纠正前一条输错的命令, 需要先下载

sudo apt install thefuck

history-substring-search

强大的历史命令搜索插件

git clone https://github.com/zsh-users/zsh-history-substring-search \
${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-history-substring-search
