如何自搭建代理服务器·基于Linux

发表于 2020-05-04 本文字数： 2.7k 标签： Linux 爬虫

爬虫因为网站限制的原因经常用到ip代理，解决方案无非几种：

  • 网络上寻找一些免费代理，优点：免费不限量；缺点：可用性较低，验证费时间费资源。一些有免费代理的网站，西刺代理，站大爷，89免费代理等等，网上可以搜出一大堆。
  • 购买代理ip，和找免费的差不多，一般有免费代理ip的网站基本都有收费的api。优点：可用性较高，较稳定。缺点：要钱。严格说，这个也不算缺点，要钱很多人也能接受，带来的方便还是很大程度的。
  • 自己搭建ip代理，自己租服务器自己搭。优点：稳定，可控。缺点：麻烦，成本高。

其实代理自己整一台服务器搭一下就可以，就是其中曲折有点多···，搭建代理也有很多种协议，工具，都实验过后选择使用tinyproxy，简单方便。

Tinyproxy的安装

Ubuntu系直接执行安装命令即可：apt install -y tinyproxy

Centos系需要先安装epel源，然后进行安装

yum -y install epel-release 
yum -y install tinyproxy

版本验证

tinyproxy -v        # tinyproxy 1.10.0

配置文件

在使用之前配置一下配置文件：vim /etc/tinyproxy/tinyproxy.conf，常用配置项：

Bind 127.0.0.1            # 允许访问服务的域名，默认只允许本地，没有限制的话注释掉即可
Port 10086                # 服务开放的端口，默认8888

开启服务

service tinyproxy start        # 开启
service tinyproxy restart        # 重启
service tinyproxy status    # 查询状态

测试代理的可用性

我的代理搭建在阿里云上，IP47.93.230.127，port10086，本地wsl使用curl代理方法测试

# -x  -- [protocol://]host[:port] Use this proxy
curl -x 47.93.230.127:10086 httpbin.org/get

错误：Can’t open PID file

错误描述：

  • tinyproxy.service: Can’t open PID file /run/tinyproxy/tinyproxy.pid (yet?) after start: No such file or directory

  • 服务start时并没有出现错误，修改了配置文件后重启服务出现该错误

分析：

  • PIDfile指定了一个用于存储服务的PID的文件路径，系统管理器在服务启动后会从这个文件中读取服务主进程的PID
  • service命令调用顺序为：ExecStart->ExecStartPost->PIDFile
  • 如果服务启动后该文件还没有写完，则会报告错误该文件不可读
  • 为了使系统管理器读取时该文件已写完，需在服务启动后加入一段延时，这时需要用到ExecStartPost=命令来调用/bin/sleep，参数为0.1。ExecStartPost=指令只会在ExecStart=调用成功后运行。

解决方案：

# 打开文件/lib/systemd/system/tinyproxy.service并在[Service]选项下添加以下内容：
ExecStartPost=/bin/sleep 0.1

# 重新加载systemd程序的配置文件
systemctl daemon-reload

# 重启tinyproxy服务
service tinyproxy restart

# 查看tinyproxy服务状态
service tinyproxy status
