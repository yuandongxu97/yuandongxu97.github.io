Docker容器技术笔记

发表于 2019-05-06 本文字数： 2.1k 标签： Docker Valine：

Docker容器是一种虚拟化技术，基于google的golang语言开发，在github进行开源，底层原理使用了Linux容器的LXC技术，并对其进行了进一步的封装。

LXC技术为将Linux进程沙盒化，使得进程之间相互隔离，并且能够控制各进程的资源分配。

与传统的虚拟化技术如VMware虚拟机等相比：

方面docker容器虚拟机
虚拟化层面模拟物理机，硬件层面基于操作系统层，软件层面
隔离级别操作系统进程
部署与启动秒级分钟级
磁盘使用基于同一个OS内核，200M以内一个虚拟机相当于一个完整的物理机，GB级
单机实例数量数千几十
宿主机系统资源/个0~0.050.1~?

应用上：

  • 虚拟机更擅长于彻底隔离整个运行环境，如云服务提供商通常采用虚拟机技术隔离不同的用户。

  • Docker通常用于隔离不同的应用，例如前端，后端以及数据库。

1. Docker优势

  1. 快速的CI、CD
  2. 内核级的虚拟化，更高的效率和性能
  3. 轻松迁移和扩展
  4. 管理方式简单

2. docker组成

  • image镜像

    docker的image是一个只读的模板，例如一个nginx image可以包含一个完整的ubuntu os，其中仅安装了nginx和其需要的依赖。

    image的来源：制作、从container save/update、repository下载

  • container容器实例

    container是docker运行独立的容器实例，是image的运行实例，可以被启动、开始、停止、删除。

    每个container都是相互隔离的，可以把其看做是一个简易版的 Linux 环境(包括root用户权限、进程空间、用户空间和网络空间等)和运行在其中的应用程序。

    image和container的关系实际上为：容器在启动的时候在镜像上创建一层可写层作为最上层，所以说容器实例是依赖image存在的，即container存在不能删除image。

  • Repository镜像仓库

    仓库(Repository)是集中存放镜像文件的场所。

    有时候会把仓库和仓库注册服务器(Registry)混为一谈,并不严格区分。实际上，仓库注册务器上往往存放着多个仓库，每个仓库中又包含了多个镜像，每个镜像有不同的标签(tag)。

    本地可以通过docker pull image从仓库下载镜像，使用docker push image将本地镜像推送到远程仓库。

3. 安装与使用

docker有社区版ce、商业ee、debain维护的io版本之分，所以根据操作系统进行版本选择安装即可。

  • windows：参考

  • debain系：如ubuntu，sudo apt install docker.io

  • Centos7

    系统版本信息：Linux centos7 3.10.0-1127.18.2.el7.x86_64 GNU/Linux

    安装依赖

    yum install -y yum-utils device-mapper-persistent-data lvm2
# yum-util 提供 yum-config-manager 功能，后两者为devicemapper驱动依赖

    安装Docker软件源（前者为官方源，后者为阿里源，二者选其一即可）

    yum-config-manager --add-repo http://download.docker.com/linux/centos/docker-ce.repo
yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

    查看安装源中的Docker-ce软件版本方法

    yum list docker-ce --showduplicates | sort -r

    安装docker-ce

    yum install docker-ce-18.03.1.ce -y        # 安装指定版本
yum install docker-ce -y        # 安装最新的stable版本

普通用户使用docker

一般来说docker只能由root和docker用户组的用户进行控制，而普通用户默认是不加入docker组的，导致普通用户操作docker会出现：Got permission denied while trying to connect to the Docker daemon socket 的错误，解决方法参考↓

# 一般docker安装后自动创建docker组，否则
groupadd docker
userchmod -aG docker yuan    # 将用户yuan添加到docker组
newgrp docker    # 刷新组信息

更换仓库源

默认源slowlyyyyy，换源是最好的解决方法，编辑docker源配置文件：vim /etc/docker/daemon.json

{
    "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"]
}

阿里云获取地址：https://cr.console.aliyun.com/cn-beijing/instances/mirrors
