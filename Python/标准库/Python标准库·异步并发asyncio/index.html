Python标准库·异步并发asyncio

发表于 2020-07-06 本文字数： 2.8k 标签： Python标准库 Valine：

Python因为GIL全局解释器锁的原因导致同一时间只有一个线程在工作，所以无法发挥多核CPU的性能，导致IO密集型的task效率非常低。

协程，Coroutine，是一种基于用户调度的线程内上下文切换技术，在操作系统中，不管是thread还是process系统开辟的，内部调用的是系统的API是抢占式的调度，而协程是非抢占式的，使用者可以在单线程的情况下使用协程进行异步编程实现高并发。

使用协程的优点：

  • 协程极高的执行效率。因为子程序切换不是线程切换且由程序自身控制，因此，没有线程切换的开销，多线程任务的线程数量越多，协程的性能优势就越明显。
  • 不需要多线程的锁机制，因为只有一个线程，也不存在同时写变量冲突，在协程中控制共享资源不加锁，只需要判断状态就好了，所以执行效率比多线程高很多。
  • 因为协程是一个线程执行，那怎么利用多核CPU呢？最简单的方法是多进程+协程，既充分利用多核，又充分发挥协程的高效率，可获得极高的性能。

asyncio是Python中的一个基于协程实现异步高并发的库，3.4版本开始引入到标准库，3.5添加了 async/await 语法进行协程的实现，其被用作多个提供高性能 Python 异步框架的基础，包括网络和网站服务，数据库连接库，分布式任务队列等等，它往往是构建 IO 密集型和高层级 结构化 网络代码的最佳选择。

1. 相关基础概念

了解一些基础概念，如阻塞和非阻塞、同步和异步、多进程和协程。

1.1 阻塞与非阻塞（任务状态）

  • 阻塞状态指程序未得到所需计算资源时被挂起的状态，程序在等待某个操作完成期间，自身无法继续干别的事情，则称该程序在该操作上是阻塞的。

    常见的阻塞形式有：网络 I/O 阻塞、磁盘 I/O 阻塞、用户输入阻塞等。阻塞是无处不在的，包括 CPU 切换上下文时，所有的进程都无法真正干事情，它们也会被阻塞。如果是多核 CPU 则正在执行上下文切换操作的核不可被利用。

  • 程序在等待某操作过程中，自身不被阻塞，可以继续运行干别的事情，则称该程序在该操作上是非阻塞的。

    非阻塞并不是在任何程序级别、任何情况下都可以存在的。
    仅当程序封装的级别可以囊括独立的子程序单元时，它才可能存在非阻塞状态。

    非阻塞的存在是因为阻塞存在，正因为某个操作阻塞导致的耗时与效率低下，我们才要把它变成非阻塞的。

1.2 同步与异步（处理逻辑）

同步：指完成事务的逻辑，先执行第一个事务，如果阻塞了，会一直等待，直到这个事务完成，再执行第二个事务，顺序执行。。。

异步：和同步相对的，异步是指在处理调用这个事务的之后，不会等待这个事务的处理结果，直接处理第二个事务去了，通过状态、通知、回调来通知调用者处理结果。

1.3 实例分析

以小明要泡茶和看电视

  • 阻塞：泡完茶再看电视
  • 非阻塞：烧水等水开的同时看电视
  • 同步：烧水、人在看着
  • 异步：烧水、人可不在，水开水壶响通知小明

1.4 协程的原理

比如一个Python爬虫task需要爬取abc三个页面也就是说该task有三个子程序（函数实例）

爬虫在爬取时因为页面数据大网速慢所以导致接收数据的时间需要3-5秒，这也就是我们所说的网络IO时间

正常情况下，爬取a - IO等待 - 爬取b - IO等待 - 爬取c - IO等待

而使用协程进行异步编程时，启动爬取后会将子程序的上下文挂起，然后启动下一个子程序，继续挂起，依此循环···，在此同时如果被挂起的上下文返回子程序完成的信号时，线程会将该子程序拿到前台进行处理，完成后继续前面的步骤。

2. asyncio库的原理与使用

asyncio提供的框架以事件循环(event loop)为中心，程序开启一个无限的循环，程序会把一些函数注册到事件循环上。当满足事件发生的时候，调用相应的协程函数。

2.1 异步方法

async/await 关键字中，async 定义一个协程函数，await 用来挂起当前并执行调用

import asyncio

# 定义
async def a():
    print("hello")
# 调用   
async def main():
    await a()

关于异步函数的执行方法

# Python3.7前
loop = asyncio.get_event_loop()
loop.run_until_complete(main())
loop.close()
# Python3.7起
asyncio.run(main())

异步函数的并发

# 1
await a()
await b()
# 2
await asyncio.gather(a(), b())

2.2 asyncio核心概念

  • EventLoop负责执行事件循环整个过程，调用各个回调函数

  • Future是回调管理器

  • Coroutine使用生成器技术来替代连续的多个回调

  • Task负责将Coroutine接口和Future、EventLoop接口对接起来，同时它自己也是一个Future

2.3 Awaitables可等待对象

可等待对象就是可以被await调用的对象，在asyncio中主要有三种：cotoutine、task、future

Coroutine，协程函数的实例

协程函数是定义形式为 async def 的函数，协程对象是调用 协程函数 所返回的对象，协程函数特点是代码块中可以将执行权限交给其他函数。

import asyncio

async def a():
    print('x')
    await asyncio.sleep(0)
    print('y')

async def main():
    await asyncio.gather(a(), a())

if __name__ == '__main__':
    asyncio.run(main())
# x
# x
# y
# y
  • await asyncio.sleep(0)用于将当前程序阻塞，也就是交出控制权
  • asyncio.gather用来并发运行任务，在这里表示协同的执行a的两个实例对象a1、a2
  • 在并发执行中，协程a1被挂起又恢复过。

Task，任务

cotoutine通过asyncio.create_task()函数被打包为Task对象，该协程将自动排入日程准备并发运行:

import asyncio

async def a():
    print('x')
    await asyncio.sleep(0)
    print('y')

async def main():
    t1 = asyncio.create_task(a())
    t2 = asyncio.create_task(a())
    await asyncio.gather(t1, t2)

if __name__ == '__main__':
    asyncio.run(main())
